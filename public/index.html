API Documentation

Welcome to the API documentation. This API provides endpoints for managing users, columns, and cards in a Kanban board application.

the base url is https://api-rest-kanban.onrender.com

  • Users

Authentication

This API uses JWT (JSON Web Tokens) for authentication. Include the token in the Authorization header of your requests as follows:

Authorization: Bearer YOUR_TOKEN

Login

Endpoint to log in and receive a JWT.

POST /login
{
  "email": "user@example.com",
  "password": "yourpassword"
}

Response:

{
  "message": "Login successful",
  "token": "YOUR_JWT_TOKEN"
}

Register

Endpoint to register a new user.

POST /register
{
  "username": "newuser",
  "email": "newuser@example.com",
  "password": "newpassword"
}

Response:

{
  "id": 1,
  "username": "newuser",
  "email": "newuser@example.com"
}

Users

Endpoints for managing users.

Delete User

DELETE /users
Authorization: Bearer YOUR_TOKEN
{
  "email": "user@example.com",
  "password": "yourpassword"
}

Response:

{
  "message": "User deleted successfully"
}

Update User Email

PUT /users/email
Authorization: Bearer YOUR_TOKEN
{
  "email": "user@example.com",
  "newEmail": "newemail@example.com"
}

Response:

{
  "message": "User updated successfully"
}

Update User Password

PUT /users/password
Authorization: Bearer YOUR_TOKEN
{
  "email": "user@example.com",
  "password": "oldpassword",
  "newPassword": "newpassword"
}

Response:

{
  "message": "User updated successfully"
}

Columns

Endpoints for managing columns.

Get All Columns

GET /columns
Authorization: Bearer YOUR_TOKEN

Response:

[
  {
    "id": 1,
    "title": "To Do",
    "board_id": 1
  },
  ...
]

Create Column

POST /columns
Authorization: Bearer YOUR_TOKEN
{
  "title": "New Column",
  "board_id": 1
}

Response:

{
  "id": 2,
  "title": "New Column",
  "board_id": 1
}

Update Column

PUT /columns/{id}
Authorization: Bearer YOUR_TOKEN
{
  "title": "Updated Column"
}

Response:

{
  "id": 2,
  "title": "Updated Column",
  "board_id": 1
}

Delete Column

DELETE /columns/{id}
Authorization: Bearer YOUR_TOKEN

Response:

{
  "message": "Column deleted successfully"
}

Cards

Endpoints for managing cards.

GET /cards/{column_id}
Authorization: Bearer YOUR_TOKEN

Response:

[

    "title": "Card 1",
    "description": "Card 1 description",
    "column_id": 1
  },
  ...

POST /cards
Authorization: Bearer YOUR_TOKEN

Update Card

  "id": 2,

Delete Card

Errors

This section describes the various error responses you might encounter when using the API.

Common Errors

400 Bad Request
{
  "error": "Invalid user data"
}
            
401 Unauthorized
{
  "error": "Access denied. No token provided."
}
            
403 Forbidden
{
  "error": "Insufficient permissions"
}
            
404 Not Found
{
  "error": "Resource not found"
}
            
500 Internal Server Error
{
  "error": "Internal server error"
}
